# firmware-png-test

Disposable smoke-test firmware for confirming that a server-rendered PNG
(from `worker/renderer/`) displays correctly on the LilyGo T5 4.7" S3 panel
and that grayscale shades are preserved through the decode path.

**This is not production firmware.** It's a standalone PlatformIO project
separate from the real firmware in `../src/`. Build, flash, look at the
screen, move on. Delete the folder when no longer needed.

## What it does

1. Embeds `worker/renderer/preview.png` as a C byte array at build time.
2. On boot, allocates a 4bpp framebuffer in PSRAM.
3. Decodes the embedded PNG scanline-by-scanline with `PNGdec`, converting
   each RGB565 pixel to 8-bit luma and writing through `epd_draw_pixel`.
4. Pushes the framebuffer to the e-paper panel once.
5. Idles forever. No WiFi, no deep sleep, no battery reading, no refresh
   loop, no overlays.

Timings are printed on the serial console (115200 baud) so you can see how
long decode and panel refresh actually take.

## One-time setup

Nothing beyond the usual `pio` tooling. Library dependencies
(`LilyGo-EPD47`, `PNGdec`) are declared in `platformio.ini` and fetched on
first build.

## Build and flash

From this directory:

```
pio run -t upload --upload-port /dev/cu.usbmodem2101
pio device monitor --port /dev/cu.usbmodem2101 --baud 115200
```

The monitor will print something like:

```
PNG display smoke test
Embedded PNG size: 32354 bytes
PNG header: 960x540, bpp=8, pixel type=2
Decode returned 1 in 412 ms
epd_draw_grayscale_image took 850 ms
Done. Frame is on screen.
```

## Updating the image

If you change `worker/renderer/src/layout.jsx` and want to test the new
rendering, regenerate the PNG and the embedded header, then reflash:

```
cd ../worker/renderer && npm run preview && cd -
python3 scripts/png_to_header.py
pio run -t upload --upload-port /dev/cu.usbmodem2101
```

The Python script has no dependencies beyond the standard library and
reads/writes default paths when run with no arguments.
